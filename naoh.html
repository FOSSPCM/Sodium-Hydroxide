The Sodium-Hydroxide Tests

This program is conducting a test of your browser's compliance with modern-day Web and Security standards. This is only a test.

About the Sodium-Hydroxide Tests

The Sodium-Hydroxide Tests are only automated tests for seeing how well your browser supports the web standards and security practices for the modern web, each organized by category. You will see a "PASS," "FAIL," or "PARTIAL" next to each test name. You want to see "PASS" for every test. If you see anything other than just a "PASS," you can find more information by accessing your browser's console (usually this is done by right-clicking, or in the case of Mac users, "secondary-clicking," this page and choosing "Inspect," then go to the "Console" tab in the box that shows up).

The name "Sodium-Hydroxide" was inspired by the name of an old, discontinued web standards test called Acid3. Sodium-Hydroxide (chemical symbol "NaOH") is a corrosive base.

Please understand that there is only so much that an automated test can do. Manual testing is needed for a more thorough test. Nevertheless, this should be good enough to convey what your browser supports in the grand scheme of things.

About the Audio and Video tests: Unfortunately, there is no reliable way to test for specific audio and video codecs other than using live files and trying to get your browser to play them; as the codec detection methods only check the IANA media type, not the underlying codec. Basically this means for codecs like AV1, H.264, Opus, and Vorbis; the tests are only checking for the container formats these codecs might wind up in, like MP4 or Ogg, but not the codecs being contained like AV1, H.264, Opus, or Vorbis.

About the Ogg Video test specifically: Towards the end of 2023 in the wake of a security vulnerability involving WebP, an image format by Google also tested here, both Google and Mozilla announced that they are dropping support for the Theora video format because it hasn't been updated since 2011 and because there are not many videos on the web that use the format at all these days (according to Mozilla, only ~0.09% of video traffic involved Theora). Since this is the only video format involving Ogg containers on the web, newer versions of Chrome, Firefox, and other modern Chromium- or Gecko-based browsers will fail the Ogg Video test. Unfortunately, Theora is historically significant (being an HTML5 recommendation at one point), there is still video on the web that use Theora, Theora has features not found in VP8 or H.264, and is the only video format reliably royalty-free (VP9 and AV1 are both attacked by patent claims); so browsers should ought to continue supporting it (Pale Moon still does), and Xiph should really update it. So, the test stays.

About the Ad/Tracker blocking test: Just to be clear, I am not using an actual ad or tracker. The test is merely trying to grab an image from a domain known to host them seeing if your browser will intercept it. Even if it doesn't, no ads or trackers will actually run. People want to protect themselves from spyware and malware, they don't like being tracked, and they don't like advertisements. The test addresses those concerns.

About the Blink element test: Yes, I know it is considered obsolete, but it is historically significant and I want it back. Obviously this can be done using CSS, but what's the fun in that?

Additional Resources and Tests

The Sodium-Hydroxide Tests are not designed to test everything about your browser, let alone anything about your internet connect. So, for your convenience, I have provided some links:

  • If you want a straight-forward understanding of what web technologies browsers support, you can visit "Can I use..." and look up a technology.
  • If you want to test your browser's speed, Speedometer 3.0 is a nice benchmark that tests the responsiveness of web browsers against simulated user interactions in a way that is more akin to real-world appllications.
  • If you want to test your internet speed, LibreSpeed is a simple Free and Open Source internet speed test.
  • If you want to see how well your browser resists fingerprinting, Cover Your Tracks by the Electronic Frontier Foundation is the best one I've seen.
  • If you want more comprehensive security and privacy tests of your browser, including IP address leaks and DNS leaks, you can find them at Browserleaks.com.
  • If you want to check how well your browser handles ICC Version 4 color profiles, there's an old test from the International Color Consortium that works pretty well, and there is a more thorough test from the developers of Little CMS as well. Make sure you test not only your browser's HTML renderer, but also your browser's PDF viewer if it has one.
  • Jamie Zawinski has a test to see if your browser can handle images of arbitrary size on his blog under a post titled "PNG antics." See if your browser can actually display the image correctly. (By "size" I mean dimensions, not bytes)

Sending feedback to your browser vendor

If you want to help your browser become more complaint and pass more tests, one way you can help is to send feature requests to the developer of you browser. The following are some of the more common browsers.

Mozilla Firefox: You can request features for Firefox at Mozilla Connect. If you are using LibreWolf, use Mozilla Connect as well. LibreWolf is built using the latest code base from Firefox.

Google Chrome: You can report an issue by clicking on the Menu, choose Help > Report an issue. Not the most intuitive way of sending feature requests, possibly because Chrome is more focused on simplicity that anything else. The Chromium issues list is a nice place for submitting feature requests for the Chromium Browser core, used not just by Chrome, but also Edge, Opera, and Brave.

Microsoft Edge: The Feedback Hub application is the simplest way to submit feature requests to Microsoft for everything including Edge, but it is only on Windows. If you are not on Windows, The Microsoft Edge Insider Community Hub is the next best thing.

Opera: Opera has a forum dedicated to suggestions and feature requests.

Brave: Brave has a forum dedicated to feature requests as well.

Apple Safari: Fill out the Feedback for for Safari and choose "Enhancement Request" from the Feedback Type dropdown menu.

Pale Moon: You can request features on The Pale Moon forum.
